# Discovery and MVP shaping

Use this file when a request is still fuzzy or when too much scope is being pulled into the first release.

## Discovery questions

- who has the problem
- how often it happens
- what workaround exists today
- what consequence comes from not solving it
- what evidence supports the request

## MVP shaping rules

- keep only the smallest slice that proves value or learning
- separate must-have outcomes from delivery polish
- protect the first release from stakeholder wish-list accumulation
- make explicit what is intentionally deferred

## Practical rule

If the first release is trying to satisfy every stakeholder equally, it is probably not an MVP.
