# User stories and metrics

Use this file when shaping backlog-ready work.

## User-story bar

A strong story should identify:

- the user or actor
- the goal
- the value of achieving it
- key acceptance criteria

If the story is too large to test or estimate well, split it.

## Acceptance criteria guidance

Good acceptance criteria:

- describe observable behavior
- cover the main success path
- call out important edge cases
- clarify constraints or permissions

## Success metrics

Pair feature work with metrics such as:

- adoption
- completion rate
- time saved
- error reduction
- retention or repeat use
- conversion or revenue influence

Choose metrics that reflect real value, not vanity activity alone.
