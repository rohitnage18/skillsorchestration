# Prioritization and roadmaps

Use this file when deciding what should happen now, next, later, or not at all.

## Prioritization lenses

Useful signals:

- user pain severity
- strategic importance
- revenue or retention impact
- confidence in the evidence
- implementation effort
- dependency unlock value
- risk reduction or compliance need

## Good prioritization behavior

- compare items against each other, not in isolation
- make tradeoffs explicit
- keep "not now" as a valid outcome
- separate urgent from important
- revisit decisions when new evidence changes the picture

## Roadmap bar

A roadmap should communicate:

- why the work matters
- what outcome each slice targets
- what is near-term versus directional
- what assumptions still need validation

Do not present a speculative wish list as if it were a committed roadmap.
